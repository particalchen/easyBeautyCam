## 一、项目概述

### 1.1 一句话描述

让不会拍照的男友/摄影师，轻松指挥模特摆出好看姿势的跨平台（iOS / Android / 鸿蒙）相机应用。

### 1.2 核心用户

- **主要用户**：拍的人（男友/摄影师），单手举手机，单手拇指操作
- **次要用户**：被拍的人（女性），未来可通过投屏实时看到相机画面

### 1.3 核心使用流程

```
打开App
  → 取景框（实时相机画面 + 白色姿势轮廓叠加）
    → 底部姿势缩略图横向滑动，切换姿势
      → 指挥模特对照轮廓摆拍
        → 按快门
          → 弹出滤镜+美颜调整浮层（默认推荐一个滤镜）
            → 可左右滑动切换滤镜 / 调整美颜参数
              → 保存 → 返回取景框继续拍
```

## 二、设计规范

### 2.1 色彩规范

| 用途 | 色值 | 说明 |
|------|------|------|
| 主色调 | `#FF8A7A` | 珊瑚粉，温暖不刺眼 |
| 主渐变 | `#FFB4A2 → #FF8A7A` | 拍照按钮渐变，浅珊瑚到深珊瑚 |
| 背景色 | `#FFFAF8` | 暖白，比纯白更温暖 |
| 深色背景 | `#1A1A1A` | 相机取景框背景（黑或深灰） |
| 文字主色 | `#2D2D2D` | 深灰，非纯黑，更柔和 |
| 文字次要 | `#999999` | 中灰，辅助信息 |
| 姿势线条 | `rgba(255,255,255,0.55)` | 白色半透明，马克笔风格 |
| 线条外发光 | `rgba(255,255,255,0.2) 1px` | 柔和边缘，增加可见度 |
| 选中态边框 | `#FF8A7A` | 主色描边 |
| 浮层背景 | `rgba(255,250,248,0.95)` | 半透明暖白背景 |

### 2.2 字体规范

| 用途 | 字体 | 说明 |
|------|------|------|
| 标题 | SF Pro Display Bold, ≥20pt | iOS 系统字体，现代感 |
| 正文 | SF Pro Text Regular, ≥17pt | 系统默认，易读 |
| 按钮文字 | SF Pro Text Medium, ≥17pt | 清晰可辨 |

| 属性 | 规范值 |
|------|--------|
| 颜色 | `rgba(255,255,255,0.55)` 白色半透明 |

| 主体轮廓 | 实线 |
| 脸朝向/手部方向引导 | 虚线（可选） |
| 叠加方式 | 覆盖在实时相机画面上方，姿势轮廓不随双指缩放变化大小 |

### 2.4 组件规范

| 组件 | 规范 |

| 滤镜横向滑轨 | 底部弹出，半透明暖白背景，横向滚动选择 |
| 美颜参数滑杆 | 竖向滑轨，磨皮/美白/瘦脸三档 |
| 镜头切换按钮 | 1x / 2x / 3x 文字按钮组 |

## 三、页面结构

### 3.1 整体架构（方案 B：单页流 + 底部工具条）

```
┌────────────────────────────────────┐
│           取景框（相机画面）           │
│  [白色半透明姿势轮廓叠加]              │
│  [姿势缩略图横向滑动条]               │
├────────────────────────────────────┤
│ [相册]  1x  2x  3x   [● 拍照]  [滤镜] │
└────────────────────────────────────┘

         ↓ 拍照后

┌────────────────────────────────────┐
│           滤镜浮层（覆盖取景框）        │
│  [关闭]       [原图] [保存]           │
│                                      │
│  [滤镜横向滑动条]                     │
│  [美颜滑杆：磨皮 | 美白 | 瘦脸]        │
└────────────────────────────────────┘
```

### 3.2 导航结构

- **单一取景框页面**（无 Tab bar）
- **姿势库**通过点击姿势缩略图右侧"展开"图标，从底部弹起全屏姿势选择器
- **相册**点击进入照片浏览（从系统相册或App专用相册读取）
- **滤镜/美颜**通过拍照后浮层处理

## 四、功能模块

### 4.1 P0 第一批（基础功能，MVP 发布）

| 功能 | 描述 | 交互 |
|------|------|------|
| **姿势轮廓叠加** | 白色半透明马克笔风格线条覆盖在相机画面上。包含：脸部轮廓及朝向示意、头部位置、手部动作、上半身姿态。线条不随相机双指缩放变化大小 | 始终覆盖在取景框上方 |
| **姿势选择** | 底部横向滑动缩略图条，点击切换姿势，姿势轮廓随之切换 | 横向滑动，点击选中 |
| **相机变焦** | 原生 1x / 2x / 3x 镜头切换按钮 | 点击按钮切换 |
| **双指缩放** | 双指 pinch 缩放相机画面，姿势轮廓保持原大小不跟随缩放 | 手势操作 |
| **即时拍照** | 点击拍照按钮，当前帧保存为照片 | 单手拇指操作 |
| **滤镜处理** | 拍完弹出滤镜选择浮层，默认选中推荐滤镜，可左右滑动切换 | 横向滑动 |
| **美颜调整** | 滤镜浮层底部有美颜滑杆（磨皮/美白/瘦脸），拍后处理，不开启实时美颜 | 竖向滑杆 |
| **照片保存** | 滤镜+美颜处理后保存到相册 | 点击保存按钮 |
| **内置姿势库** | App 内置 4 个姿势（手工绘制的白色轮廓 PNG），启动时从远程地址自动下载更多姿势到本地，支持增量更新 | 随 App 预置 4 个 + 启动时下载 |
| **姿势远程获取** | App 启动时从指定的远程地址获取姿势列表并下载到本地，支持姿势增量更新 | 网络获取，本地缓存 |

### 4.2 P1 第二批（用户自定义）

| 功能 | 描述 |
|------|------|
| **自定义姿势导入** | 用户可从相册导入自己准备的姿势图片，加入自己的姿势模板库 |
| **我的姿势管理** | 查看/删除/分类自己添加的姿势 |
| **姿势分类收藏** | 用户可对姿势进行收藏/分组 |

### 4.3 P2 第三批（社交+智能）

| 功能 | 描述 |
|------|------|
| **姿势分享** | 用户可将自建姿势发布到社区供他人使用 |
| **姿势导入** | 从他人分享的姿势中挑选加入自己的模板库 |
| **场景识别推荐** | 识别环境（海边/室内/街景等），自动推荐对应姿势 |
| **被拍者监看** | 未来支持投屏到对方设备，被拍者实时看到相机画面 |

## 五、技术架构

### 5.1 跨平台框架

**Flutter** — 单套代码支持 iOS / Android / 鸿蒙三大平台

| 层次 | 技术选型 | 说明 |
|------|---------|------|
| 框架 | Flutter | 跨平台主力 |
| 状态管理 | Riverpod | 轻量、可测试 |
| 相机 | `camera` 插件 + `google_mlkit_pose_detection` | 实时预览 + 姿态骨架 |
| 图片处理 | `image` 包 + `photo_editor` 类插件 | 滤镜+美颜本地处理 |
| 导航 | `go_router` | 轻量路由 |
| 本地存储 | `Hive` | 姿势模板/用户数据本地持久化 |
| 远程姿势获取 | `http` + `path_provider` | 启动时从远程地址下载姿势 PNG 到本地 |
| 平台发布 | App Store / 华为应用市场 / 鸿蒙应用商店 | 未来分发 |

### 5.2 架构分层

```
┌─────────────────────────────────┐
│         UI Layer (Flutter)        │
│  CameraPreview / PoseOverlay     │
│  FilterSlider / BeautyPanel     │
├─────────────────────────────────┤
│       Business Logic Layer       │
│  PoseManager / FilterChain      │
├─────────────────────────────────┤
│         Platform Layer           │
│  Camera API / Image Processing  │
│  (via Platform Channel / Plugin) │
└─────────────────────────────────┘
```

**核心原则**：
- 业务逻辑和 UI 分离
- 平台特性（相机/美颜/滤镜）封装成 Platform Channel / 插件，不直接写在 UI 层
- 滤镜和美颜走 GPU/本地处理，不上传云端

### 5.3 姿势模板格式与获取

**格式**：
- `PNG` 半透明图片（白色线条，带 Alpha 通道）
- 命名规则：`pose_[场景]_[序号].png`（如 `pose_outdoor_01.png`）
- 存放位置：`resources/` 目录（手工姿势图）+ `assets/poses/`（下载的姿势）

**数量**：
- 内置 4 个（手工提供，用户自行制作），存放于 `resources/` 目录
- 启动时从远程地址自动下载更多姿势到本地（增量更新）

**远程获取机制**：
- App 首次启动及每次启动时，向指定远程地址请求姿势列表 JSON
- 下载新的姿势 PNG 到本地目录，覆盖或增量添加
- 使用 `Hive` 缓存上次同步状态，避免重复下载

### 5.4 滤镜实现（本地）

预设滤镜基于 Color Matrix 或 LUT 实现，全部本地处理，无需网络。

## 六、交互细节

### 6.1 取景框交互

| 操作 | 行为 |
|------|------|
| 点击姿势缩略图 | 切换姿势轮廓覆盖 |
| 横向滑动姿势条 | 浏览更多姿势 |
| 双指 pinch | 缩放相机画面，姿势轮廓保持原大小 |
| 点击 1x/2x/3x | 切换镜头焦距 |
| 点击拍照按钮 | 保存当前帧 |
| 点击相册按钮 | 打开已保存照片浏览 |
| 点击滤镜按钮 | 打开滤镜选择（需要先有照片） |

### 6.2 滤镜浮层交互

| 操作 | 行为 |
|------|------|
| 左右滑动滤镜条 | 切换不同滤镜，预览实时应用效果 |
| 拖动美颜滑杆 | 调整美颜参数（磨皮/美白/瘦脸） |
| 点击保存 | 处理完成，保存到相册 |
| 点击关闭/返回 | 丢弃当前处理，返回取景框 |

### 6.3 单手操作保证

- 拍照按钮位于屏幕下方中央或偏右，单手拇指可达
- 主要操作（拍照、滤镜切换）均可单手完成
- 无需双手配合的复杂手势（双指缩放除外，这是通用手势）
